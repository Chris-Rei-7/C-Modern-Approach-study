2. C99 数组指示器 (Designated Initializers) 总结
1.核心作用与语法
指示器允许你在初始化数组时，明确指定具体下标（索引）的值。未被指定的元素会自动被编译器初始化为 0。

传统痛点：要给第 3 个和第 15 个元素赋值，中间要写一大堆 0，极易数错位数。
int a[15] = {0, 0, 29, 0, 0, 0, 0, 0, 0, 7, 0, 0, 0, 0, 48};

指示器写法：直接用 [下标] = 值 的形式，清晰明了。

C
int a[15] = {[2] = 29, [9] = 7, [14] = 48}; 
2.三大重要特性

 顺序自由 (Order Independent)
你不需要按照下标从小到大的顺序来写。打乱顺序完全不影响结果，这在修改代码时非常方便。

C
int a[15] = {[14] = 48, [9] = 7, [2] = 29}; // 完全合法
 自动推断长度 (Length Inference)
如果在声明数组时省略了方括号里的长度（即 [] 内留空），编译器会去寻找指示器中最大的那个下标，并据此推断出数组的长度（最大下标 + 1）。

C
int b[] = {[5] = 10, [23] = 13, [11] = 36}; 
// 编译器看到最大下标是 23，自动将数组 b 的长度设定为 24。
 混合使用 (Mixing Styles)
你可以把“传统逐个赋值”和“指示器赋值”混合在同一个花括号里使用。跟在指示器后面的元素，会顺着该指示器的下标继续往下排。

C
int c[10] = {5, 1, 9, [4] = 3, 7, 2, [8] = 6};
/* 结果相当于：
   c[0] = 5, c[1] = 1, c[2] = 9 (传统顺序)
   c[3] = 0 (跳过了，自动为0)
   c[4] = 3 (指示器赋值)
   c[5] = 7, c[6] = 2 (顺着 [4] 往后排)
   c[7] = 0 (跳过了)
   c[8] = 6 (指示器赋值)
   c[9] = 0 (末尾未指定，自动为0)

3. 运算符 sizeof 可以确定数组的大小（字节数）。如果数组 a 有 10 个整数，那么 sizeof(a)通常为 40（假定每个整数占 4 字节）。
还可以用 sizeof 来计算数组元素（如 a[0]）的大小。用数组的大小除以数组元素的大小可以得到数组的长度：
sizeof(a) / sizeof(a[0])
当需要数组长度时，一些程序员采用上述表达式。例如，数组 a 的清零操作可以写成如下形式：
for (i = 0; i < sizeof(a) / sizeof(a[0]); i++)
 a[i] = 0;
