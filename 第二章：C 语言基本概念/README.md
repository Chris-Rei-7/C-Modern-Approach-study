1.`.c` 文本文件不能直接运行，必须经过“三道工序”才能变成真正可执行的机器码：

 1. **预处理 (Preprocessing)**：专门处理以 `#` 开头的指令（比如 `#include`），负责在正式编译前修改或补充代码。
 2. **编译 (Compiling)**：把预处理过的高级代码，翻译成机器能看懂的“目标代码（机器指令）”。注意：这时候还不能运行！
 3. **链接 (Linking)**：。把我的目标代码和系统自带的库函数（比如 `printf`）像拼图一样拼装在一起，生成最终的 `.exe` (Windows) 或可执行文件 (Linux/Unix)。

 - **避坑预警**：以后写代码如果报错，要分清是“编译错误”（语法写错了）还是“链接错误”（比如函数名拼错了，链接器找不到对应的库）。

7.定义常量的名字

一、 定义常量的目的

提升可读性：当程序中存在具有特定业务含义的数值常量时，直接使用数字会降低代码的可读性。通过为常量命名，可以清晰地表明该数值的具体含义，便于后续的代码阅读与维护。

二、 宏定义（Macro Definition）的基础语法

指令格式：使用预处理指令 #define 来为常量命名。

代码示例：#define INCHES_PER_POUND 166

语法注意：#define 属于预处理指令（与 #include 类似），并非普通的 C 语言执行语句，因此行末不需要且不应添加分号。

三、 宏定义的工作原理

预处理替换：在程序正式进行编译之前，预处理器会介入。它会在源代码中查找所有的宏名称，并将其精确替换为宏所代表的实际值。

执行效果：例如，代码中的 weight = (volume + INCHES_PER_POUND - 1) / INCHES_PER_POUND; 在预处理后，编译器实际处理的语句等价于 weight = (volume + 166 - 1) / 166;。

四、 进阶用法：定义表达式

功能扩展：宏不仅能用于定义单一的数值常量，还可以用来定义复杂的表达式。

代码示例：#define RECIPROCAL_OF_PI (1.0f / 3.14159f)

安全规范：当宏定义的内容包含运算符时，必须使用括号将整个表达式包裹起来。这是一种防御性编程习惯，可以有效避免宏在代码中被替换时，因上下文的运算符优先级问题而引发难以察觉的计算逻辑错误。

五、 宏的命名规范

大写原则：宏的名称应全部使用大写字母。

行业共识：需要明确的是，全大写并非 C 语言编译器的强制语法要求。然而，这是全球 C 程序员遵循了数十年的标准编码规范。在任何实际开发中，都应严格遵守此约定，以便区分宏与普通的变量。
