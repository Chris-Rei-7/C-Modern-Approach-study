1.核心概念区分：实参 vs 形参

实际参数 (Arguments / 实参)：是你在调用函数时，真实递交给函数的数据。比如 average(x, y) 里的 x 和 y。

形式参数 (Parameters / 形参)：是函数在内部定义时，用来接住外来数据的“内部变量”或“占位符”。比如图文中提到的 a 和 b。

2.最重要的灵魂机制：按值传递 (Pass by Value) * 图文中有一句极其关键的话：“把变量 x 和 y 的值复制给形式参数 a 和 b”。

这意味着，C 语言默认是以“给复印件”的方式在传递数据。函数拿到的只是 x 和 y 数值的复印件（存放在内部的新变量 a 和 b 里）。因此，无论函数内部怎么折腾、修改 a 和 b，都绝对不会影响到外面原本的 x 和 y 变量。这是一种极其重要的数据保护机制。

3.实参的极度自由形式

传递给函数的实参不一定非得是纯粹的变量名。只要最终计算出的数据类型匹配，它可以是任何合法的表达式：

直接传具体的常数：average(5.1, 8.9)

传复杂的算术表达式：average(x/2, y/3) （计算机会先在外面默默算好 x/2 和 y/3 的结果，然后再把这两个最终结果的复印件传进函数里）。

4.返回值的“就地替换”思维

如果一个函数有返回值（比如算出了平均数），你可以直接把这串“函数调用代码”当成一个普通的值，塞进其他语句里。

比如 printf("Average: %g\n", average(x, y));。程序走到这里时，会先跳去执行 average 函数，算出结果（假设是 4.5），然后把 average(x, y) 这一整串代码“就地替换”成 4.5，最后再交给 printf 打印出来。这让代码极其紧凑优雅。

 编译器在调用前遇到原型。就像使用赋值一样，每个实际参数的值被隐式地转换成相应
形式参数的类型。例如，如果把 int 类型的实际参数传递给期望得到 double 类型数据

7. 关于数组型实际参数的另一个重要论点是，函数可以改变数组型形式参数的元素，并且
改变会在相应的实际参数中体现出来。例如，下面的函数通过在每个数组元素中存储 0 来修改
数组：
void store_zeros(int a[], int n)
{
 int i;
 for (i = 0; i < n; i++)
 a[i] = 0;
}
函数调用
store_zeros(b, 100);
会在数组 b 的前 100 个元素中存储 0。数组型实际参数的元素可以修改

8. 形式参数是多维数组，声明参数时只能省略第一维的长度。例如，如果修改sum_array 函数使得 a 是一个二维数组，我们可以不指出行的数量，但是必须指定列的数量

9. 在 C99 中，可以使用复合字面量来避免该问题，复合字面量是通过指定其包含的元素而创建的没有名字的数组。下面调用 sum_array 函数，第一个参数就是一个复合字面量：
total = sum_array((int []){3, 0, 3, 4, 1}, 5);
在这个例子中，复合字面量创建了一个由 5 个整数（3、0、3、4 和 1）组成的数组。这里没有对数组的长度做特别的说明，其长度是由复合字面量的元素个数决定的。当然也可以显式地指明长度，如(int[4]){1, 9, 2, 1}，这种方式等同于(int[]){1, 9, 2, 1}。
一般来说，复合字面量的格式如下：先在一对圆括号内给定类型名，随后是一个初始化器，用来指定初始值。因此，可以在复合字面量的初始化器中使用指示器（ 8.1 节）一样，
而且同样可以不提供完全的初始化（未初始化的元素默认被初始化为 0）。例如，复合字面量(int[10]){8, 6}有 10 个元素，前两个元素的值为 8 和 6，剩下的元素值为 0。
函数内部创建的复合字面量可以包含任意的表达式，不限于常量。例如：
total = sum_array((int []){2 * i, i + j, j * k}, 3);
其中 i、j、k 都是变量。复合字面量的这一特性极大地增加了其实用性。
复合字面量为左值（ 4.2 节），所以其元素的值可以改变。如果要求其值为“只读”，可以在类型前加上 const，如(const int []){5, 4}。
